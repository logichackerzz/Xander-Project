- 連續踩坑：pandas 2.2.2 在 Render 上一直編譯失敗
  - 原因一：requirements.txt 裡 numpy 排在 pandas 後面，pandas metadata 生成需要 numpy
  - 原因二：Render 預設 Python 3.14，pandas 2.2.2 沒有 3.14 的 wheel，強制編譯 C++ 卻不相容
  - 解法方向：在 Render Environment Variables 加 `PYTHON_VERSION=3.11.9`，等明天驗證

## 反思

- 卡住的地方：Render 的 Python 版本偵測機制，`runtime.txt` 沒有被正確讀取，比預期多花了很多時間
- 問題 → 解法：`cpython-314` 出現在 log → 確認是 Python 3.14 問題 → 改用 Environment Variable 強制鎖版本
