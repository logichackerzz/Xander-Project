# Session 002 — 2026-04-22

## 主題
修正 Claude Code 啟動目錄問題（PowerShell 環境）

## 問題
每次開啟 Claude Code 都停在 `C:\Users\mybab` 而非 `C:\Xander-agent`。

## 診斷過程
- 上次的 `~/.bashrc` 修正對 PowerShell 完全無效（.bashrc 只適用 Git Bash）
- 使用者終端機為 **PowerShell**
- PowerShell 設定檔路徑：`C:\Users\mybab\OneDrive\Documents\WindowsPowerShell\Microsoft.PowerShell_profile.ps1`
  （Documents 資料夾在 OneDrive，非一般路徑）
- 執行原則：`RemoteSigned`（可正常載入本機設定檔）

## 解決方式
在正確的 PowerShell 設定檔中加入 `claude` wrapper function：

```powershell
function claude {
    Set-Location 'C:\Xander-agent'
    & "C:\Users\mybab\.local\bin\claude.exe" @args
}
```

## 結論
設定完成。之後每次在 PowerShell 輸入 `claude`，會自動切換至 `C:\Xander-agent` 再啟動 Claude Code。

## 關鍵資訊
- Claude 執行檔：`C:\Users\mybab\.local\bin\claude.exe`
- PowerShell 設定檔：`C:\Users\mybab\OneDrive\Documents\WindowsPowerShell\Microsoft.PowerShell_profile.ps1`
- `.claude` 設定資料夾位置：`C:\Users\mybab\.claude`（正常，不需移動）

## 補充 — 同日後續設定

### 主題
新增 `cc` 捷徑與 NO_FLICKER 設定

### 做了什麼
- 在 PowerShell 設定檔加入 `$env:CLAUDE_CODE_NO_FLICKER = "1"`，消除 Claude Code 啟動時的畫面閃爍
- 新增 `cc` function，與 `claude` 完全相同，讓使用者可以用更短的指令啟動

### 最終設定檔內容
```powershell
$env:CLAUDE_CODE_NO_FLICKER = "1"

function claude {
    Set-Location 'C:\Xander-agent'
    & "C:\Users\mybab\.local\bin\claude.exe" @args
}

function cc {
    Set-Location 'C:\Xander-agent'
    & "C:\Users\mybab\.local\bin\claude.exe" @args
}
```

### 結論
開新終端機後，直接輸入 `cc` 即可啟動 Claude Code 並自動切換至工作目錄。
