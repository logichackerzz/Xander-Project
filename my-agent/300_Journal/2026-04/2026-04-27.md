今天安裝了 `/brainstorm` 這個 Skill。

一開始我以為它是「先建立結構再開始思考」，但後來發現理解有點偏。它真正強調的是：在 AI 動手之前，先透過問答讓我跟 AI 對齊方向——確認我真正想要的是什麼，而不是讓 AI 猜完就開始做，做完才發現方向錯了。

跟 Claude Code 內建的 Plan Mode 相比，Plan Mode 是「AI 先列計劃再執行」，`/brainstorm` 多了前面一層「釐清需求、比較方案、確認共識」，更像是執行前的對焦會議。

這個邏輯其實適用在很多地方——不只是跟 AI 合作，跟人合作也一樣。先花時間對齊，比做到一半發現方向不對要省力得多。

## 散戶投資儀表板 — 專題啟動

今天開始規劃一個新的專題：散戶投資人儀表板。

目標是做一個**有質感、有別於市面工具**的網站，給課堂評分用。三個核心功能：財報爬取、多市場資產負債表、市場情緒工具。

最大的收穫是做了一個清醒的技術棧選擇。一開始我有點猶豫要不要用 Streamlit——快速、Python 全搞定——但它的版型就是「工具感」，沒辦法做出有質感的東西。最後選了 Next.js + FastAPI 的組合，我完全不會 JavaScript，但 Claude 說他可以負責全部的程式碼，我只需要下指令、測試、告訴他哪裡不對。

這個分工模式很有趣：我是產品經理 + 測試員，AI 是全端工程師。之前我一直把 AI 當「查資料的工具」，這次感覺更像是真的在「協同開發」。

另一個決策是資產負債表的輸入方式選了「手動 + 圖片 OCR」——上傳對帳單截圖，讓 Claude API 自動解析出持倉。這個功能本身就是個亮點，對課堂展示來說差異化很明顯。
