# 期末發表規劃 — Folio × Claude Code 協作框架

> 截止：2026 年 6 月初｜上台時間：15-20 分鐘（未確認）｜評審：老師 + 業界混合

---

## 核心定位

> **「散戶的資訊是破碎的。我用 Folio 整合它。我用框架設計讓 AI 幫我做到這件事。」**

兩大主軸，缺一不可：
- **主軸 A：Folio** — 產品是什麼、為什麼叫 Folio、它解決什麼問題
- **主軸 B：協作方法論** — Claude Code 跟其他 AI 哪裡不同、我設計的記憶/技能/路由系統

目標感受：同學眼睛發光、頭抬起來；教授即使知道你用了 AI，還是被吸引。

---

## 敘事弧（三幕劇）

### 第一幕：命名懸念
「我想跟你們講一個名字的故事。」
- Folio = 莎士比亞第一版印刷全集的一張薄頁
- Portfolio = 投資組合
- 這個產品，把散落各處的資訊，收進了同一本冊子裡

### 第二幕雙軌並進
**軌道 A（產品）：** 問題 → Demo → 功能 → 技術
**軌道 B（方法論）：** 反問「你只是叫 AI 幫你做？」→ 展示系統設計

### 第三幕：收束
> 協作不是依賴，是放大。
