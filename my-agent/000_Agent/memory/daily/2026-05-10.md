### 1. 跑馬燈（Ticker）大幅升級

**起點**：原本三列跑馬燈（美股 / 台股 / ETF）只有靜態 mock 數據，卡片 64px。

**過程討論：**
- 加入商品類：黃金（`GC=F`）、白銀（`SI=F`）、原油（`CL=F`）到美股那條
- 卡片大小反覆調整：64 → 72 → 80px，容器從 440px → 560px → 1120px
  - 我主動提出「1120px 在 1440px 螢幕上會溢出」的設計建議，Xander 同意後嘗試
  - 最終決定卡片 80px、容器 1120px（後來因為其他佈局問題留著）
- 移除三列的「美股 / 台股 / ETF」標籤 pill
- 加入即時價格顯示（原靜態 `p` 欄位）

**即時價格 API 串接：**
- 後端新增 `GET /api/market/prices?symbols=...` endpoint（`market.py`）
- 第一版用 Yahoo Finance v7 batch API → 回傳 401（已加認證要求）
- 改用 `yfinance` + `ThreadPoolExecutor`（10 workers 並行）一次拉 38 支
- 前端加 `yf` 欄位對應 yfinance symbol（台股加 `.TW`、興櫃加 `.TWO`）
- 前端新增 `livePrices` state + `useEffect` 批次 fetch，靜態數據當 fallback

**Port 問題插曲：**
- 後端原本跑在 8002，reload 沒吃到新 endpoint（404）
- 舊 process（PIDs 2272 / 30448）用 bash `kill` 和 `taskkill` 無法結束（Windows 系統限制）
- 解法：改用 port 8003 另起一個乾淨的 server
- 全前端 `localhost:8002` 改成 `localhost:8003`（用 `sed` 批次替換）

**台股跑馬燈調整（Xander 親自提供清單）：**
- 移除：富邦金、國泰金、兆豐金

### 2. Hero 頁面設計討論：Slogan 定位之路

這是今天花最多時間的討論，反覆迭代了將近十次。

**文字方向確認：**
- 原始：「財報是企業對外說話的語言 / 我們是你的財務翻譯機」
- 討論要加入「Folio」字樣
- 我給出 A/B/C/D 四個方向：
  - A 對稱式、B 動詞強調、C 話中有話、D 宣言式
- Xander 偏好 B，想融合 B+C
- 我提出多個融合版本，Xander 最終自己合成：
  - **「財報是企業的語言 / 難讀，但不該難懂 / Folio 就是為此而生」**
- 調整為一行：「財報是企業的語言，難讀，但不該難懂——Folio 就是為此而生」
- 再移除標點後：「財報是企業的語言 難讀 但不該難懂 — Folio 就是為此而生」

**位置反覆嘗試：**

| 嘗試位置 | 問題 | 決策 |
|----------|------|------|
| 英文標題正下方 | 字體 Lora 跟 Urbanist 衝突，字太小 | 改字型、放大 |
| 統一換 Urbanist | `font-light` 在中文渲染怪 | 改 `font-normal` |
| 底部左側（CTA 上方） | 擠壓右側跑馬燈空間 | 固定 200px 寬 |
| CTA 上方 / 下方互換 | 按鈕全寬後 Xander 說「怪」 | 改回 self-start |
| 加膠囊包住文字 | 我提醒太 AI 感，Xander 同意 | 拿掉膠囊 |
| 最終：英文標題正下方純文字 | — | **定案** |

**最終 slogan 樣式：**
- `fontFamily: var(--font-urbanist)`、`font-normal`
- `fontSize: clamp(0.82rem, 1.1vw, 1rem)`
- `mt-2`、`pl-8`（縮兩格）
- 純文字，無容器

### 5. Hero 入場動畫系統

**打字機效果插曲：**
- 我實作了完整的 `useTypewriter` hook，headline 完成後觸發 slogan
- Xander 問「會不會太怪？」
- 我直接說：會。打字機在大標題上節奏慢、力道分散、品牌調性不符
- 拿掉，改做 clip-path 掃入

**Clip-path 掃入實作：**
- 兩行分別包 `motion.span`，各自做 `inset(0 100% 0 0)` → `inset(0 0% 0 0)`
- 第二行 delay 0.14s，形成閱讀節奏
- 修正：`leading-[1.0]` 讓 descender（g、y 字母）被裁切 → 各行加 `pb-[0.15em]`
- 全部掛在 `ready` state 上（intro 結束才觸發）

### 6. 頁面三段滾動架構

**架構設計：**
- Section 1（Hero，h-screen）
- Section 2（Dashboard 儀表板）
- Section 3（彩蛋頁，h-screen）

**滾動機制：**
- wheel 事件 + `scrollIntoView`，一格跳一段
- Section 1 離場：heroOpacity + heroOverlay（白色遮罩隨 scroll 淡入）

**關鍵 Bug 發現（今天最重要的技術發現）：**
- `window.scrollY` 永遠是 0，`useScroll()` 也失效
- 原因：LayoutShell 的 `<main>` 有 `overflow-auto`，scroll container 不是 window
- 解法：wheel handler 改監聽 `main` element，用 `main.scrollTop`
- `useScroll` 改為 `useScroll({ container: mainRef })`

**Section 2 動畫修正：**
- `whileInView` 在頁面載入時就偷偷 trigger，用戶滾進去時動畫早已結束
- 改用 `IntersectionObserver` + `dashboardInView` state 控制
- `cardVariants` 加入 scale：`y: 56, scale: 0.95 → y: 0, scale: 1`
- Section 2 Header「Folio」字也做 clip-path 掃入，與 Section 1 同一套風格

**Section 3 彩蛋頁：**
- 內容：五段文字，每段淡出非關鍵字，「Folio」保持 indigo 發光脈動
- `eggInView` state + IntersectionObserver 控制播放時機
